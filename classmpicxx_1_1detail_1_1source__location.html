Detailed Description

Represents information of a specific source code location.

Example usage:

#include <iostream>
#include <string_view>
#include <mpicxx/detail/source_location>
void log(std::string_view message,
const mpicxx::detail::source_location& loc = mpicxx::detail::source_location::current())
{
std::cout << "info:\n"
<< " " << location.file_name() << "\n"
<< " " << loc.function_name() << "\n"
<< " " << loc.line() << "\n\n"
<< message << std::endl;
}
int main() {
// normal usage
log("Hello, world!");
// better function name
log("Hello, world!", mpicxx::detail::source_location::current(MPICXX_PRETTY_FUNC_NAME__));
return 0;
}

Constructs a new mpicxx::detail::source_location with the respective information about the current call side.

The MPI rank is set to std::nullopt if an error occurred during the call to MPI_Comm_rank (an exception is thrown or a return code different than MPI_SUCCESS is returned).

static std::string mpicxx::detail::source_location::stack_trace ( [[maybe_unused] ] const int  max_call_stack_size = 64)
inlinestatic

Returns the current stack trace.

For a better stack trace (precise function names) the linker flag -rdynamic is set if and only if any MPICXX_ASSERTION_... has been activated during Cmake's configuration step.

A sample stack trace (while -rdynamic is set) could look like:

stack trace:
#5 ./output.s: test(int) [+0x3]
#4 ./output.s: foo() [+0x1]
#3 ./output.s: main() [+0x1]
#2 /lib/x86_64-linux-gnu/libc.so.6: __libc_start_main() [+0xe]
#1 ./output.s: _start() [+0x2]
Parameters
[in]max_call_stack_sizethe maximum depth of the stack trace report
Returns
the stack trace nodiscard
Attention
The stack trace report is only available under GCC and clang (to be precise: only if __GNUG__ is defined). This function does nothing if __GNUG__ isn't defined.
