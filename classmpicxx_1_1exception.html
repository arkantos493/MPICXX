Detailed Description

The base class of all exceptions in the mpicxx namespace.

The mpicxx::detail::source_location class is used to provide more information in case of an exceptional case. 

     It uses a [`std::shared_pointer<std::string>`](https://en.cppreference.com/w/cpp/memory/shared_ptr) for the what message
     to be able to provide a **noexcept** copy constructor and copy assignment operator (as requested by the C++ standard).

mpicxx::exception::exception ( const detail::source_location &  loc = detail::source_location::current())
inline

Construct an exception, i.e. construct the exception location message.

If the location message couldn't be constructed, the respective exception gets directly caught to prevent a call to std::terminate during stack unwinding.

Prints a detailed stack trace if MPICXX_ENABLE_STACK_TRACE has been enabled during the CMake configuration step.

void mpicxx::exception::prepend_to_what_message ( T &&  msg)
inlineprotected

Tries to prepend msg to the current message.

There are two possible cases:

  1. the initial message (containing the detail::source_location message) was created successfully: try to prepend msg; if an exception is thrown, this function has no effect
  2. the initial message (containing the detail::source_location message) was not created successfully: try to store msg; if an exception is thrown, this function has no effect
