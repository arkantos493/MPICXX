Detailed Description

Provides more verbose assert alternatives, supporting MPI ranks.

Author
Marcel Breyer
Date
2020-07-19
Copyright
This file is distributed under the MIT License.

The asserts are currently separated into three levels:

  • 0: no assertions are enabled (default)
  • 1: assertions that check preconditions, i.e. conditions that must hold for the function to complete successfully
  • 2: additionally to the assertions of level 1, assertions that check if the parameters make sense in the current context, but which are not required for the function to complete successfully (e.g. increment of a past-the-end iterator), are added

During CMake's configuration step, it is possible to enable a specific assertion level using the -DMPICXX_ASSERTION_LEVEL option.

Builtin assertion syntax and example output:

assert(("Parameter can't be negative!", n > 0));
output.s: ./example.cpp:42: void test(int): Assertion `("Parameter can't be negative!", n > 0)' failed.

Mpicxx assertion syntax and example output:

MPICXX_ASSERT_PRECONDITION(n >= 0, "Parameter can't be negative!: n = {}", n);
// alternative: MPICXX_ASSERT_SANITY
PRECONDITION assertion 'n >= 0' failed
on rank 1
in file ./example.cpp
in function 'int test(int)'
@ line 42
Parameter can't be negative!: n = -1
stack trace:
#7 ./output.s: test(int) [+0x3]
#6 ./output.s: foo() [+0x1]
#5 ./output.s: bar() [+0x]
#4 ./output.s: baz() [+0x]
#3 ./output.s: main() [+0x1]
#2 /lib/x86_64-linux-gnu/libc.so.6: __libc_start_main() [+0xe]
#1 ./output.s: _start() [+0x2]

For a meaningful stack trace to be printed the linker flag -rdynamic (GCC and clang) is added if and only if the MPICXX_ASSERTION_LEVEL is greater than 0.
MSVC currently doesn't support a stack trace.

In addition the assertions call MPI_Abort if the assertion is executed within an active MPI environment, std::abort otherwise.

#define MPICXX_ASSERT_PRECONDITION (   cond,

Defines the MPICXX_ASSERT_PRECONDITION macro if and only if the MPICXX_ASSERTION_LEVEL, as selected during CMake's configuration step, is greater than 0.

This macro is responsible for all precondition checks. If a precondition of a function isn't met, the respective function isn't guaranteed to finish successfully.

An example could be to check whether an iterator can be safely dereferenced or not.

Parameters
[in]condthe assert condition
[in]msgthe custom assert message
[in]...varying number of parameters to fill the {fmt} like placeholders in the custom assert message
#define MPICXX_ASSERT_SANITY (   cond,
  msg,
  ... 
)

Defines the MPICXX_ASSERT_SANITY macro if and only if the MPICXX_ASSERTION_LEVEL, as selected during CMake's configuration step, is greater than 1.

This macro is responsible for all sanity checks. If a sanity check isn't successful, the respective function can still complete, but the result isn't necessarily meaningful.

An example could be the check whether an attempt is made to increment a past-the-end iterator.
