Detailed Description

template<bool is_const>
class mpicxx::info::iterator_impl< is_const >

Provides iterator and const_iterator for an info object.

The standard reverse_iterator and const_reverse_iterator are provided in terms of std::reverse_iterator<iterator> and std::reverse_iterator<const_iterator> respectively.

All necessary functions for a random access iterator are implemented.

Each iterator can be in one of three possible states:

  1. singular iterator (the iterator is currently not attached to any info object)
  2. iterator referring to an info object referring to MPI_INFO_NULL
  3. valid

The presence of 1. and 2. can be checked using assertions if the assertion categories SANITY and PRECONDITION are activated during CMake's configuration step.

Template Parameters
is_constif true a const_iterator is instantiated, otherwise a non-const iterator

template<bool is_const>
using mpicxx::info::iterator_impl< is_const >::value_type = std::conditional_t<is_const, std::pair<const std::string, const std::string>, std::pair<const std::string, proxy>>

std::iterator_traits value type that can be obtained by dereferencing the iterator.

In case of a non-const iterator, the value will be returned by a info::proxy object to allow changing its value.

In case of a const_iterator, the value will directly be returned as const std::string because changing the value is impossible by definition.

template<bool is_const>
using mpicxx::info::iterator_impl< is_const >::pointer = pointer_impl<value_type>

std::iterator_traits pointer type to the [key, value]-pair iterated over.

Because it is not possible to simply return value_type* (dangling pointer to a local object), it is necessary to wrap value_type in a pointer_impl object (which in turn overloads operator->).

template<bool is_const>
reference mpicxx::info::iterator_impl< is_const >::operator[] ( const difference_type  n) const
inline

Get the [key, value]-pair at the current iterator position + n.

If the current iterator is a const_iterator, the returned type is a std::pair<const std::string, const std::string>.

If the current iterator is a non-const iterator, the returned type is a std::pair<const std::string, proxy>, i.e. the [key, value]-pair's value can be changed through the proxy object.

Parameters
[in]nthe requested offset of the iterator (n may be negative)
Returns
the [key, value]-pair ([[nodiscard]])
Precondition
*this must not be a singular iterator.
*this must not refer to an info object referring to MPI_INFO_NULL.
*this must be dereferenceable, i.e. the position denoted by the current iterator + n must be in the half-open interval [0, nkeys), where nkeys ist the size of the referred to info object.
